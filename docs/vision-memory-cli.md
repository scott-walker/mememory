# mememory — Product Vision

## One binary, zero friction

`mememory` is the single entry point for the entire mememory product. It replaces manual Docker commands, config editing, and shell scripts with a unified cross-platform Go binary.

## Installation

```bash
# Linux / macOS
curl -fsSL https://get.mememory.dev/install.sh | sh

# macOS
brew install mememory

# Windows
winget install mememory
```

One command installs the `mememory` binary. No dependencies except Docker.

## User journey

```bash
# First time setup — one command does everything
mememory setup

# What setup does:
# 1. Pulls and starts Docker stack (PostgreSQL + Ollama + Admin)
# 2. Waits for healthchecks
# 3. Downloads embedding model
# 4. Auto-detects MCP client (Claude Code, Claude Desktop, etc.)
# 5. Writes MCP server config to the right file
# 6. Configures SessionStart bootstrap hook
# 7. Prints "Ready. Start a new Claude session."
```

After setup, the user starts a new Claude session and memory just works. No manual configuration.

## Commands

| Command | What it does |
|---------|-------------|
| `mememory setup` | First-time setup: Docker stack + MCP config + hooks |
| `mememory bootstrap` | SessionStart hook: auto-detect project, print memories |
| `mememory status` | Health check: Docker services, DB connectivity, model loaded |
| `mememory stop` | Stop Docker stack |
| `mememory start` | Start Docker stack |
| `mememory export` | Export all memories as JSON |
| `mememory import` | Import memories from JSON |
| `mememory upgrade` | Pull latest Docker images, run migrations |
| `mememory uninstall` | Remove Docker stack, optionally remove data |

## Bootstrap (SessionStart hook)

```bash
mememory bootstrap
```

1. Runs natively on the host (no Docker exec)
2. Detects project from working directory (`git rev-parse --show-toplevel` → basename)
3. Calls admin HTTP API (`localhost:4200/api/memories`)
4. Formats and prints Markdown to stdout
5. Override: `mememory bootstrap --project myapp --persona reviewer`

Hook config (auto-generated by `mememory setup`):
```json
{
  "hooks": {
    "SessionStart": [{
      "matcher": "",
      "hooks": [{ "type": "command", "command": "mememory bootstrap" }]
    }]
  }
}
```

## Architecture

```
mememory (host)              Docker stack
─────────────────              ────────────
setup ──────────────────────> docker compose up
bootstrap ──────HTTP──────────> admin API (:4200) ──> PostgreSQL
status ─────────HTTP──────────> admin API (:4200)
                                     │
                                  Ollama (embeddings)
```

- CLI = thin client, no state, no DB access
- Docker stack = all infrastructure
- Communication = HTTP to admin API (already exposed)

## Build & Distribution

- `CGO_ENABLED=0` — static binary, no libc dependency
- Single binary per platform: linux/amd64, linux/arm64, darwin/amd64, darwin/arm64, windows/amd64
- GitHub Releases with checksums
- Homebrew tap, winget manifest
- Install script for curl-pipe-sh flow

## Principles

- **Zero config by default.** Setup auto-detects everything. Manual config is optional override.
- **Cross-platform.** Go binary works on Linux, macOS, Windows. No shell scripts in the critical path.
- **Docker for infra, native binary for UX.** Heavy services (DB, embeddings) run in Docker. User-facing CLI runs natively for speed and OS integration.
- **Graceful degradation.** If Docker is not running, CLI prints clear error with fix command. If admin API is unreachable, bootstrap exits silently (agent starts without memory, not crashes).
